Connections BVT Suite

This package includes both the GUI and API BVT tests

Download

Download all the bvt.zip file, ane unzip it into a directory. You should see the following:

  • README.html - this file
  • bvt.gui.jar - the executable jar package
  • bvt.api.jar - the API test suite, and data population tool.
  • bvt.adm2 - the Connectiosn wsadmin test suite.
  • lib - libraries required to run the tools.
  • logs - log files goes to here
  • resource - the files used during test, such as attachments
  • test_config - sample test configurations

GUI BVT

Spedify the server/browser info, and selecte tests to run

Make a copy of test_config/testTemplate_PREBVT.xml to testConfig.xml, and open edit the new file , simply follow the comments inside itself. You can set the server URL, and your browser type, as well as what test suite you need to run.

Run GUI Tests:

java -jar bvt.gui.jar testConfig.xml
This will lauch the browser you specified in testConfig.xml and do the tests.

Test result report are in test-output directory, after run.

API BVT and the Data Population Tool

This is the new API framework, not the Janet tool. 

java -jar bvt.api.jar http://yourserver.com/
 * Test Users are recorded in ./resources/ProfileData_{BVTserver}.properties inside bvt.api.jar
 * Get a properties file, such as ProfileData_lc45linux1.properties, as sample.  Create a ProfileData_{yourserver}.properties
 * ProfileData_{yourserver}.properties should be under ./resources inside or outside bvt.api.jar
 * For AS Search API, modify ./resources/ASSearch_crawler.properties for Connections admin user/password
This will run all the tests and data population against the specified server.
