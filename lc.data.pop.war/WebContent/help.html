Getting started with Login

Back

  2. Url field of the login block
  3. username for the login block
  4. Password field for the login block
  5. Another use for the login block
The login block is the starting point for all data population scripts. This block represents a user logging in to the url provided. All other blocks must be placed inside of a login block. Another use of the login block is create something as a different user, as shown by arrow 5. In the example shown above, if we were to just attach the replys directly to the entry without first wrapping them in a login, it would appear as though fadams had created all the replies. Anything placed inside of a login block will look like it was created the user specifed, until you place another login block.

Adding aditional member slots

Back
  1. The block modifier button
  2. Member slots
By default, components that can be assigned members come with one member slot. By clicking the blue button in the corner of the block, you can add extra member slots.


In order to add extra member slots, drag the member blocks into the activity block. When you click the change button, you will be returned to the main script composition screen.

You will now see extra member slots in the block.

Duplicating blocks

Back
One of the more useful features of this site is the ability to duplicate blocks. This allows you to quickly create repeated information. To duplicate a block, right click on the block. This brings up a context menu.

Clicking on the duplicate option will create a copy of the selected blocks.

The Community block

Back
The Community block is different from the others because it can contain other types of blocks. It is possible to add bookmarks and forum topics and replies to a community, and they will automatically be applied to the built in forum and book mark community widgets.

Known Issues

Back
  • Sometimes, when you click to edit a field, the text gets highlighted but you can't type. Click out of the field and then click back in and wait a second for it to catch up, it should let you type now.
  • When you import a template, sometimes it doesn't show up until you click or scroll in the workspace window.
  • When naming an export, if the "Export To File" button is disabled (greyed out), change the name and click outside the text box and somewhere else onto the page to reenable the "Export To File" button.
